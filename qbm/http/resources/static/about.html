Framework Overview

QB HTTP is a modern C++17 library for building high-performance, asynchronous HTTP/1.1 and HTTP/2 clients and servers. It's designed to integrate seamlessly with the QB Actor Framework or work standalone with qb-io.

Key Features

  • Modern C++17 - Uses latest language features for clean, efficient code
  • Asynchronous I/O - Non-blocking operations for high performance
  • HTTP/1.1 & HTTP/2 - Full support for both protocol versions
  • Middleware System - Extensible request/response processing pipeline
  • Advanced Routing - URL patterns, parameters, and wildcard matching
  • Static File Serving - Built-in support with compression and caching
  • File Upload - Multipart form-data parsing and handling
  • JSON Support - Native JSON request/response handling
  • Authentication - JWT and custom auth middleware
  • Validation - Request validation with JSON Schema-like rules

Architecture

Core Components

  • Router - Central request routing and dispatch
  • Context - Request/response container with lifecycle management
  • Middleware - Composable request processing components
  • Session - Connection management and protocol handling
  • Controllers - Class-based route organization

Performance Characteristics

  • Event-driven architecture using libev
  • Zero-copy operations where possible
  • Efficient memory management with RAII
  • Lock-free data structures for inter-core communication
  • Optimized HTTP parsing with llhttp
  • Header compression with HPACK (HTTP/2)

Command Line Examples

# List all uploaded files
curl http://localhost:8080/api/files

# Upload a file with description
curl -X POST \
  -F "file=@example.txt" \
  -F "description=My test file" \
  http://localhost:8080/api/upload

# Get file metadata
curl http://localhost:8080/api/files/upload_1234567890_example.txt

# Delete a file
curl -X DELETE http://localhost:8080/api/files/upload_1234567890_example.txt
